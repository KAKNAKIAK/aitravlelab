1. LLM 추론(LLM Inference) 개념

LLM 추론이란?

"AI가 생각하고 답하는 과정"

AI가 방대한 데이터를 학습한 경험을 바탕으로 질문의 의도를 파악하고 가장 적절한 답변을 생성하는 과정입니다.

비유로 이해하기

특정 분야의 전문가가 자신의 지식과 경험을 총동원하여 질문에 답하는 것과 유사합니다.

3. GPT 프로젝트(Projects) 기능

핵심 개념

"나만의 디지털 작업 서랍"

특정 주제나 업무와 관련된 모든 자료를 한곳에 모아 관리하고 작업할 수 있도록 돕는 기능입니다.

활용 목적

"복잡한 자료를 다루는 장기 프로젝트"에 적합합니다.

실습 단계

1

새 프로젝트 생성

특정 업무 단위로 프로젝트를 생성합니다.

예: '2025년 3분기 신제품 런칭'

2

자료 업로드

해당 프로젝트와 관련된 다양한 참고 자료를 업로드합니다.

예: 보고서, 경쟁사 분석 파일 등

3

맞춤 지침 설정

프로젝트 내에서 GPT가 역할을 수행하는 방식을 정의합니다.

예: "이 프로젝트에서 너는 신제품 마케팅 매니저의 역할을 수행하며, 모든 답변은 긍정적이고 설득력 있는 톤으로 작성해 줘."

4

자료 기반 대화

업로드된 자료와 설정된 지침에 따라 GPT와 대화합니다.

예: "업로드된 모든 자료를 종합하여 신제품의 핵심 타겟 고객 페르소나를 3가지 타입으로 정의해 줘."

4. GPT 빌더(Custom GPT) 기능

핵심 개념

"특정 임무에 특화된 AI 전문가 자체를 만드는 기능"

코딩 없이 대화만으로 특정 임무에 특화된 맞춤형 AI를 직접 생성할 수 있는 강력한 기능입니다.

활용 목적

"특정 반복 업무를 자동화"하는 데 최적화되어 있습니다.

실습 단계

Create 탭 (대화로 쉽게 만들기)

빌더와의 대화를 통해 AI의 기본 역할을 정의합니다.

예시: "우리 회사 인사 규정과 복지 규정에 대해 정확하게 답변해 주는 챗봇을 만들고 싶어."

Configure 탭 (직접 세부 설정하기)

이름/설명

AI의 이름과 간략한 설명을 설정합니다.

예: '인사팀 규정 도우미', '사내 규정에 대해 쉽고 정확하게 안내합니다.'

지침(Instructions) ⭐ 가장 중요

AI 전문가의 행동 강령을 매우 구체적으로 명시합니다.

예: "너는 회사의 친절한 인사팀 직원이다. 직원의 질문에 대해 업로드된 규정 파일에 근거해서만 답변해야 하며, 추측성 답변은 절대 하지 않는다. 답변 끝에는 항상 관련 규정의 조항 번호를 명시해 줘."

지식(Knowledge)

AI가 참조할 '교과서' 역할을 할 파일을 업로드합니다.

예: '최신 인사 규정집.pdf'

기능(Capabilities)

웹 검색 기능 등을 켜거나 끔으로써 AI의 정보 접근 범위를 제한합니다.

예: 웹 검색 기능 끔

테스트 및 저장/공유

미리보기 창에서 직접 질문하며 AI의 답변이 의도대로 동작하는지 테스트하고, 만족스럽다면 저장하여 링크를 통해 동료들과 공유할 수 있습니다.

5. 프로젝트 vs GPT 빌더 비교

프로젝트

🔬 연구실

"다양한 자료를 펼쳐놓고 탐색하며 결과물을 만들어가는 연구실과 같습니다."

  • 복잡한 자료 다루기
  • 장기적인 작업
  • 탐색적 분석
  • 결과물 생성

GPT 빌더

🤖 자동화 로봇

"정해진 임무를 반복적으로 완벽하게 수행하도록 만든 자동화 로봇과 같습니다."

  • 반복 업무 자동화
  • 전문화된 AI
  • 일관된 답변
  • 팀 공유 가능
